🔐 Security & Validation

All operations are gated by three security layers:

· wrangler.toml – KV namespace and environment definitions.
· stripe_shell_init.sh – System hardening and dependency verification before any network call.
· monti_auth_validation.sh – OAuth token validation and rate‑limit checks for every API request.

All sensitive information (keys, passwords, private keys) is never stored – it is ephemerally loaded from encrypted environment variables during the Termux execution.

💰 Financial Incentive Model

The workforce operates on a proof‑of‑value consensus:

1. Data Providers – Users who share neuromorphic or telecom data receive a share of the processed value.
2. Compute Nodes – Cloud providers earn a transaction fee for each successful cash‑out or analysis.
3. Governance – Decisions on donation rates and treasury allocation are voted by active nodes via dash.montidroid.com/governance.

The newDonationRate (currently 0.85 ETH) is dynamically adjusted based on network activity and real‑world market conditions, ensuring sustainable passive income for participants.

⚙️ Setup & Usage

Prerequisites

· Termux (Android) or any POSIX shell
· Node.js ≥ 20
· Cloudflare account with Workers & KV
· Stripe, AWS, and GCP credentials (optional for full multi‑cloud)

Installation

```bash
git clone https://github.com/montinode/.github-2
cd .github-2
npm install
```

Run the Termux Cash‑Out (Local)

```bash
node montinode_private_termux.js
```

This will:

· Read portfolio.csv (if populated)
· Estimate total value in USD
· Log the execution to monti_termux_log.txt
· Simulate routing to JOHNCHARLESMONTI.COM

Deploy the Full Workforce

```bash
# Cloudflare Workers
wrangler deploy

# GCP Cloud Run
gcloud run deploy monti-workforce --source .

# AWS Lambda
aws lambda update-function-code --function-name MontiWorkforce --zip-file fileb://function.zip
```

After deployment, the workforce auto‑registers with dash.montidroid.com and begins processing incoming data.
